Introduction to cout

You can make so many different programs in C++. But in order to do that, something that really helps and is sometimes even vital to a program is printing to the terminal/console/command line. It may seem hard or overwhelming, but it is one of the easiest things you can do in C++. To print to the terminal, you must use the cout object. The cout object takes anything that you choose to enter and prints it to the console. An example of how to print the words Hello World! to the console is on the right.



Basics of cout

Cout comes from the iostream library. Libraries are groups of pre-compiled code that are used in multiple different programs. To use these libraries, you must include them at the top of your C++ file. The iostream library contains many more objects than just cout, but we'll get into those later. For now, all you need to know is that in order to use cout, you need to include the iostream library.

Using cout

As we've talked about in the last lesson, cout is very important to a program, and you'll use it for many, many, many reasons! From displaying text for the user to read all the way to displaying variables for debugging a program. So how can we use it? Well, lets look back at our image from the previous lesson and really break down that line of code!



Breakdown of the line

Observe the line on the left. It may look a little scary and complex, but don't worry, it isn't at all!



Breakdown of std::

Let's start with the std:: part of this cout line. Std in C++ means "Standard", and it represents the C++ standard library. But wait! Then shouldn't it be iostream::cout << "Hello World!"; because cout is from the iostream library? Well, not exactly. The reason that it is std:: and not iostream::is because std is something called a namespace, while iostream is, as discussed in the last lesson, a library. A namespace is essentially a box that holds all sorts of variables, functions, etc. It helps keep things organized and easier to use. When we include the iostream library inside of our file, we are fundamentally taking everything inside of the iostream library and dumping it into the box, that box being the std namespace. As we just said earlier, though, std represents the C++ standard library. That means that the namespace std holds everything in the C++ standard library.

Whew! That was a lot to grasp onto. Don't worry if you are struggling to understand everything that was just said. We've just covered a lot that most other demos or courses wouldn't usually cover within the first couple of lessons.



Breakdown of cout

Next up, we have the actual cout part of this cout line! Cout, as stated in the last lesson, is an object of the iostream library. This is the actual thing that will be printing the text to the console. There isn't much more to cover, since a lot of it was explained in the last lesson. So let's move onto the insertion operator!



Breakdown of Insertion Operator (<<)

Wait... what? Insertion operator? What the heck is that? Odd name, right? Well, the insertion operator is something used to give data to an output stream. It is represented with two less-than symbols like this: <<. This is mandatory for cout because this is how you actually give the text to cout. Without it, you can't talk with cout and won't be able to send your text to it.

One thing to be careful of while using the insertion operator, though, is not to mix it up with it's brother, the extraction operator. Similar to how you give data to cout through the insertion operator, the extraction operator allows you to get or recieve data from something. It is what we use to get user input, which we will talk more about in the next unit. The reason why we say to be careful of mixing these two up is because it is represented with two greater-than symbols: >>, which can become easy to confuse with the insertion operator's two less-than symbols. Make sure you don't do this because you don't want to take data from cout, you want to give data to cout!



Breakdown of the input

Ah... Here it is! Here is the point in the line where what is in it is what is printed out to the console. In this image on the left, we have Hello World! in quotation marks. This is very important to have these quotation marks because it is what tells the program that it is a sentance or a group of multiple characters. Whenever you want to print words or specific text out, you must wrap it in quotes.

But what about debugging as we mentioned earlier? Glad you asked!!! Let's say that you create a new varaible in your code. You modify it a couple of times in the code and now you have unfortunately lost track of what the variable's value is. Here comes cout to save the day! You can print a variable to the console using cout! You have do it differently than you would print text to the console, but it's still really simple. How you print a variable to the console is instead of typing quotation marks like we did for the text, you just type the name of the variable. For example, if you had a variable named test and it was set to be equal to 3, then you can print that by typing std::cout << test;. Cout will then print out the number 3 because that is what the variable is equal to. The reason you can't use quotation marks when printing a variable is because cout will think that you want to print the actual name of the varible. If you had typed std::cout << "test";, then it wouldn't print out 3 like before, it would print out "test" because cout sees anything in quotes as a group of characters that need to be printed to the console.



Breakdown of the semicolon

Finally! We reach the end of the line! The semicolon. Now, although it doesn't seem like it does much, the semicolon is one of the most important things of ANY C++ line, beyond the cout line. What the semicolon does is it marks the end of the line; Oops! Must have just put that one there by habit! But as you can see, it is similar to a period in English. A period in English marks the end of a sentance so that a new one can start. The program recognizes the semicolon and knows that it can stop reading the line there as is.
