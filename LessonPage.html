Introduction to cout

You can make so many different programs in C++. But in order to do that, something that really helps and is sometimes even vital to a program is printing to the terminal/console/command line. It may seem hard or overwhelming, but it is one of the easiest things you can do in C++. To print to the terminal, you must use the cout object. The cout object takes anything that you choose to enter and prints it to the console. An example of how to print the words Hello World! to the console is on the right.



Basics of cout

Cout comes from the iostream library. Libraries are groups of pre-compiled code that are used in multiple different programs. To use these libraries, you must include them at the top of your C++ file. The iostream library contains many more objects than just cout, but we'll get into those later. For now, all you need to know is that in order to use cout, you need to include the iostream library.

Breakdown of the line

Breakdown of std::

Let's start with the std:: part of this cout line. Std in C++ means "Standard", and it represents the C++ standard library. But wait! Then shouldn't it be iostream::cout << "Hello World!"; because cout is from the iostream library? Well, not exactly. The reason that it is std:: and not iostream::is because std is something called a namespace, while iostream is, as discussed in the last lesson, a library. A namespace is essentially a box that holds all sorts of variables, functions, etc. It helps keep things organized and easier to use. When we include the iostream library inside of our file, we are fundamentally taking everything inside of the iostream library and dumping it into the box, that box being the std namespace. As we just said earlier, though, std represents the C++ standard library. That means that the namespace std holds everything in the C++ standard library.

Whew! That was a lot to grasp onto. Don't worry if you are struggling to understand everything that was just said. We've just covered a lot that most other demos or courses wouldn't usually cover within the first couple of lessons.



Breakdown of cout

Next up, we have the actual cout part of this cout line! Cout, as stated in the last lesson, is an object of the iostream library. This is the actual thing that will be printing the text to the console. There isn't much more to cover, since a lot of it was explained in the last lesson. So let's move onto the insertion operator!



Breakdown of Insertion Operator (<<)

Wait... what? Insertion operator? What the heck is that? Odd name, right? Well, the insertion operator is something used to give data to an output stream. It is represented with two less-than symbols like this: <<. This is mandatory for cout because this is how you actually give the text to cout. Without it, you can't talk with cout and won't be able to send your text to it.

One thing to be careful of while using the insertion operator, though, is not to mix it up with it's brother, the extraction operator. Similar to how you give data to cout through the insertion operator, the extraction operator allows you to get or recieve data from something. It is what we use to get user input, which we will talk more about in the next unit. The reason why we say to be careful of mixing these two up is because it is represented with two greater-than symbols: >>, which can become easy to confuse with the insertion operator's two less-than symbols. Make sure you don't do this because you don't want to take data from cout, you want to give data to cout!



Breakdown of the input

Ah... Here it is! Here is the point in the line where what is in it is what is printed out to the console. In this image on the left, we have Hello World! in quotation marks. This is very important to have these quotation marks because it is what tells the program that it is a sentence or a group of multiple characters. Whenever you want to print words or specific text out, you must wrap it in quotes.



Breakdown of the semicolon

Finally! We reach the end of the line! The semicolon. Now, although it doesn't seem like it does much, the semicolon is one of the most important things of ANY C++ line, beyond the cout line. What the semicolon does is it marks the end of the line; Oops! Must have just put that one there by habit! But as you can see, it is similar to a period in English. A period in English marks the end of a sentence so that a new one can start. The program recognizes the semicolon and knows that it can stop reading the line there as is.

Variables

Variables are a HUGE part of programming. Variables are things that store data and can be changed. We won't get too much into variables now, but just remember that image on the left, because you might see it again in a couple of units (foreshadowing...). Observe that image, though. You can see a pink clump of data. Let's say for this example that that data is a number, maybe 4, as a random number. Right now, that 4 isn't stored anywhere, meaning it's of no use to us. But if we store that number somewhere, say in a variable, then we can access it and manipulate it whenever we need to. That's why the image shows data in the bucket that says "variable". It represents us putting our data into a variable to store it for later, whenever we need it. You'll also see an arrow that is going from the inside of the bucket to the outside the bucket. That represents how we access the data that is stored inside of our varaible, since we can take what's in the bucket and change it or view it.



Variables and cout

Now that you know a little bit more about what variables are, lets talk about how to actaully print them to the console. Look at the image on the right. You'll see that it looks almost identical to the line that we worked with last lesson (std::cout<< "Hello World!";). The only difference here is that we aren't using quotation marks. Before, we wanted to type out a set of characters, Hello World! But now we want to print out the value that the variable is storing. The reason why we don't use quotation marks is because C++, when it sees a pair of quotation marks, will see what's inside of it as a set of characters. This means that if we tried saying std::cout<< "test"; with test being the name of our variable, the program would print out the word test. This is not our goal though, since we are trying to print what is inside of test. So instead we try typing std::cout<< test; and now we have success! The program has printed out the value that test is storing.

Concatenation

Concatenating is a very powerful and useful tool we can use with many things in C++. We're gonna try it with couting in this lesson. Concatenation is when you take multiple different things and put them all together to form one sequential thing, like a chain of links. How do you think we can do this with the cout line in C++? If you said adding multiple insertion operators (<<), then you'd be correct! By doing this, we can mix together printing out text (quotation marks) and variables (non-quotation marks). How does that look, though? How would you write that cout line?



Concatenating, couting, and the \n

Well, the answer to that question is in the image on the left. Let's analyze it. So it looks like we have a variable called name, and it's storing the value "John Doe". Then in the next line we have our cout line that is printing out the phrase "Hello, " and then printing out the name variable. Then we have another set of characters in the quotation marks at the end, which are "!\n". This would result in: Hello, John Doe! Something that you probably noticed was that orange-tinted \n piece. That \n piece stands for new line. It has to be used in quotation marks, as well. The way that C++ sees the \n is as a line break, so it starts a new line to continue printing things out to the console. This is very helpful because if you had a couple of cout lines like this:

std::cout << "Hello";
std::cout << "World!";

Then the program would print HelloWorld! to the console. The reason is because there is no \n, or line break. But if we typed:

std::cout << "Hello\n";
std::cout << "World!";

Then the program would print:

Hello
World!
